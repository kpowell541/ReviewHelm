Two Variants, One Layout

Review and Polish sessions share the same component structure. The differences:

Review — filtered by stack, accent color is violet

Polish — no stack filter (universal checklist), accent color is green

Review shows stack name + icon in the header. Polish shows the mode name.

Navigation

Back button returns to Home (Polish) or Stack Select (Review).

Header includes mode icon and title with subtitle for context.

Route: /review/sessions?stack=go and /polish/sessions

New Session Button

Prominent CTA at the top — creates a new session and immediately navigates into it.

Uses the mode's accent color as background for visual consistency with the mode cards on Home.

Active Session Cards

New additions vs current:

1. Progress bar — visual indicator of items completed vs total

2. Percentage shown on the right in accent color

3. "X of Y items" replaces just "X items reviewed"

4. Date shown in the top-right corner

Progress bar uses gradient from the mode's accent color.

Completed Session Cards

Visually subdued — title text is lighter (#9898b0 vs #e4e4ec).

Green checkmark badge with final coverage percentage replaces the plain "✅" emoji.

Subtle green border tint (#10b98122) to indicate completion.

Tapping opens the Session Summary screen rather than the checklist.

Reopen button: Amber button on each completed card. Marks session as incomplete and navigates back into the checklist so users can revise verdicts or continue reviewing.

Section Labels

Uppercase section labels ("ACTIVE", "COMPLETED") with count badges.

Count badges help users quickly see how many sessions are in each state.

Empty State

When no sessions exist, show a centered illustration with encouraging text.

Different messaging per mode: "Start your first review!" vs "Start polishing your next PR!"

Deletion

Long-press triggers an Alert.alert confirmation dialog.

Hint text at the bottom reminds users of this interaction.

Future enhancement: swipe-to-delete with react-native-gesture-handler.
