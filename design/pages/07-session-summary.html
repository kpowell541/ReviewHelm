Purpose

Report card shown after completing a checklist session. Provides an at-a-glance summary of performance and highlights areas for follow-up.

Navigated to automatically when "Complete Session" is tapped on Screen 6.

Score Cards

Three top-level metrics:

Coverage — % of items reviewed. Green ≥ 70%, amber 50-69%, red < 50%.

Confidence — average across all rated items. Same color scale as Gaps (1-2 red, 3 amber, 4-5 green).

Issues — total "Attention" verdicts. Always red to draw the eye.

Issues by Severity

Breakdown of the issue count by severity level: Blocker (red), Major (amber), Minor (blue), Nit (gray).

Matches the severity chip colors from the checklist session screen.

Items Needing Attention

List of items with "needs-attention" verdict. Each row shows a severity-colored dot and the item text.

Tapping any row navigates to Deep Dive (Screen 8) for that item.

Learning Opportunities

Items where the user rated confidence ≤ 2, sorted lowest first.

Each row shows a confidence badge (red, matching the Gaps color system).

Tapping navigates to Deep Dive for focused learning.

Session AI Cost

NEW. Per-session cost display showing:

Total cost: Large green number in the header.

Stats: Call count, token count, model used.

Feature breakdown: Cost attributed to Tutor Chat, Comment Draft, and Learn Mode with individual call counts.

Budget context: Small card below showing how this session's spend fits within the monthly budget. Uses the same green→amber gradient bar from Settings.

Uses estimateCost() from pricing.ts. Per-call usage tracked via recordUsage().

PDF Export

Violet button at the bottom. Generates HTML → PDF via expo-print, then shares via native share sheet.

If "Auto-Export PDF" is enabled in Settings, triggers automatically on navigation to this screen.

PDF report includes: scores, issues, items needing attention, session cost.
