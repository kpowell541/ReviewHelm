Go · Error Handling
Verify context cancellation is handled correctly in goroutines
😕 Conf 2/5
Context cancellation in goroutines is about making sure your concurrent code can be stopped cleanly. When you launch a goroutine, it runs independently. If the caller's context gets cancelled (timeout, user abort, parent done), the goroutine needs to know — otherwise it becomes a goroutine leak. The key pattern is using select with ctx.Done():
select {
case <-ctx.Done():
  return ctx.Err()
case result := <-ch:
  // process result
}
This ensures the goroutine exits promptly when the context is cancelled.

Purpose

Focused learning mode for weak areas. Shows one concept at a time from the user's gaps (confidence ≤ 3). AI generates a lesson explanation, then offers practice exercises and deep dive access.

Navigated to from Learn Stack Select (Screen 5, right panel).

Progress Bar

Green gradient bar showing position in the gap queue (e.g., "1 of 5").

Items sourced from getWeakestItems(20) filtered to confidence ≤ 3 for the selected stack.

Item Card

Green left border (learn mode accent). Shows:

Breadcrumb: Stack · Section

Item text: The concept to study

Confidence badge: Current confidence with emoji, using standard color system (1-2 red, 3 amber)

Stats: Severity level and session count

Before Starting

Two choices:

"Start Learning with Sonnet": Green button. Calls AI to generate a concept explanation. Model name shown in button text.

"View base content instead": Text link. Opens Deep Dive (Screen 8) for offline content. No API call needed.

If no API key configured, shows a prompt to visit Settings.

During Lesson

AI-generated explanation appears as a chat bubble with code examples. Item card compacts to save space.

"Practice exercise": Green outline button. Sends a follow-up prompt asking the AI for a hands-on exercise.

"Open AI Tutor chat": Navigates to Deep Dive's AI Tutor tab for free-form follow-up questions.

"Next item →": Advances to next gap. Clears current messages. Only visible if more items remain.

Empty State

Shown when the selected stack has no gaps (all items confidence ≥ 4). Celebration emoji with "No gaps to study!" message and back button.
