Purpose

Helps draft professional PR review comments. User describes what they found, AI generates a well-structured comment with code suggestions, then the user can refine and copy it.

Navigated to from the checklist session's "Found an issue — tap to draft a comment" nudge.

Item Context Card

Blue left border. Shows which checklist item the comment is for, with stack/section breadcrumb.

Provides context to the AI so the generated comment is relevant.

Input Flow

"What did you find?": Optional freeform text describing the specific issue. More detail = better generated comment.

Generate button: Shows the model name ("Generate Draft with Sonnet"). Disabled if no API key. Spinner while generating.

Draft Output

Generated comment displayed in an editable card. Can include inline code blocks rendered in JetBrains Mono.

The draft is directly editable — user can tweak wording before copying.

Refinement Actions

📋 Copy: Copies to clipboard. Shows "✅ Copied!" feedback. Also auto-saves to session.

✂️ Shorter: Sends "Make this more concise" instruction to AI.

📝 More detail: Sends "Make more detailed with a code suggestion" instruction.

🤝 Softer tone: Sends "Make the tone softer and more encouraging" instruction.

💾 Save to session: Saves draft to the current review session (green, only visible when in a session).

🔄 Regenerate: Discards current draft and generates fresh from original context.

Rendered Preview

Shows what the comment will look like as formatted markdown — bold text, code blocks with syntax highlighting, etc.

Helps verify the comment reads well before pasting into GitHub/GitLab.

Cost Efficiency

Uses the smallest role-specific output cap (600 tokens for comment-drafter role) to keep costs low.

Refinements reuse the existing context, avoiding full regeneration cost.

Response caching serves identical drafts for the same item + context.
