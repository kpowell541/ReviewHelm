Purpose

Central dashboard for tracking knowledge gaps discovered during review/polish sessions.

Items appear here when users rate their confidence low (1-2) on checklist items. As they practice and improve, items move through Active → Improving → Mastered.

Tapping any gap card opens the Deep Dive screen (Screen 8) for that topic.

Summary Strip

Three count chips at the top give an instant overview:

Red Active — confidence 1-2, needs work

Amber Improving — confidence 3, trending up

Green Mastered — confidence 4-5, previously a gap

New addition — the current implementation only shows Active and Improving.

Gap Card Design

Confidence number: Replaces emoji with a numeric badge (1-5) in a tinted circle. More scannable and accessible than emojis.

Item text: Shows the actual checklist item text via findItemById(), truncated with ellipsis.

Stack tag: Small pill showing which stack (Go, JS/TS, Java) the item belongs to.

Session count: How many sessions this item has appeared in.

Trend indicator: Shows direction of change. "↑ was X" (green) for improvement, "↓ was X" (red) for regression. Helps users spot items slipping back.

Three Sections

Active Gaps: Confidence 1-2. Red-tinted confidence badges. Sorted by confidence level then session count.

Improving: Confidence 3 with upward trend. Amber-tinted. Shows where the user came from ("was 1").

Mastered: Confidence 4-5, items that were gaps. Green-tinted, slightly dimmed. Celebrates progress. "Review" button opens Deep Dive to revisit the topic — if confidence drops after re-rating, the item moves back to Active.

Current implementation only has Active and Improving. Mastered section is new.

Data Source

All data from useConfidenceStore:

getWeakestItems(20) returns items sorted by learning priority.

Each item has: itemId, stackId, currentConfidence, trend, learningPriority, ratings[].

Item text resolved via findItemById(gap.itemId).
