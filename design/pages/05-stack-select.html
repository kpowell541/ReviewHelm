Purpose

Entry point for both Review and Learn flows. User picks a tech stack before starting a session or studying gaps.

Same base layout, different data: Review shows confidence averages, Learn shows gap/improving counts.

Stack Cards

Left border: Colored per stack — orange Java, blue JS/TS, green Go, violet Polish.

Icon: Stack emoji in a rounded square. 44×44px container.

Title + description: From STACKS registry in checklistRegistry.ts.

Bottom row: Item count + section count on the left, PDF export button on the right.

Review Mode Badges

Shows confidence average (e.g., "4.2") in a violet pill on each card.

Calculated from getSectionAverages() in useConfidenceStore. Only shown if the user has rated items in that stack.

Tapping a card navigates to /review/sessions?stack={id}.

Learn Mode Badges

Red badge: count of items with confidence ≤ 2 (active gaps).

Green badge: count of items at confidence 3 with upward trend (improving).

Badges hidden if zero. If no gaps at all, shows "No gaps yet" message.

All Stacks card: Green-accented special card at the bottom. Shows total gap count. Routes to /learn/all.

PDF Export

Export PDF button: Amber-styled button on every stack card. Generates a printable PDF of the full checklist for that stack.

PDF includes: stack title, all sections, all checklist items with their text, and empty checkboxes for offline use.

Uses expo-print to generate HTML → PDF, then expo-sharing to share/save.

Tapping the export button does not navigate — only the card body navigates to the session flow.
